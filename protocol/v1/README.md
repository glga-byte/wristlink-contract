# WristLink Message Contract v1

WristLink v1 messages are compact JSON-compatible maps intended for Garmin
Connect IQ app-message transport.

## Phone-to-Watch Envelope

Every phone-to-watch envelope contains:

- `v`: protocol version, currently `1`
- `id`: 26-character ULID string generated by the phone
- `kind`: one of `point`, `timer`, `note`, or `command`
- `createdAt`: UTC ISO 8601 timestamp
- `ttlSec`: optional positive expiration interval in seconds
- `payload`: kind-specific JSON object

The serialized JSON envelope must fit within 1024 UTF-8 bytes.

## Payload Kinds

- `point`: latitude/longitude point with optional label and note
- `timer`: countdown timer with a label and duration in seconds
- `note`: short textual note with optional title
- `command`: named command with optional JSON-object arguments

## Watch Acknowledgements

Acknowledgements reference the original phone message with `ackFor` and use one
of these statuses:

- `accepted`
- `rejected`
- `unsupported`
- `retryable`

The phone send queue treats `accepted` as sent, `rejected` and `unsupported` as
failed, and `retryable` as pending when the original message kind requires
watch-level acknowledgement.
